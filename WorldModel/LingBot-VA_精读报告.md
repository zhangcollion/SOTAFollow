## 1 项目概述

### 1.1 研究背景与核心问题

传统 Vision-Language-Action（VLA）模型采用前馈范式直接**映射观测到动作**，存在四大根本性缺陷：

- **表征纠缠**：单一网络需同时学习视觉理解、物理动力学与电机控制，导致样本效率低、泛化能力差
- **因果性缺失**：块级双向扩散允许未来令牌影响过去预测，违背物理世界的时间因果性
- **长时记忆不足**：块级生成无跨块持久缓存，长时任务易出现时间漂移与 "失忆"
- **实时性瓶颈**：视频生成计算密集，同步推理导致控制延迟过高，无法满足实时机器人要求

### 1.2 整体技术路线

采用 "**先预测世界演化，再反推动作**" 的世界建模范式，将视频与动作统一建模为自回归序列：

1. **视觉动力学预测**：基于历史观测与动作，通过自回归扩散预测未来 K 帧视觉隐状态

2. **逆动力学解码**：从预测的视觉状态转换中，解码出对应 τK 个可执行动作

   逆动力学回答 "**想要发生这件事该做什么动作**"；让环境从当前状态$z_t$变成我们想要的目标状态$z_{t+1}$；应该执行什么动作$a_t$才能实现这个目标

3. **闭环控制**：每执行完一个动作块，用真实观测更新模型状态，修正预测轨迹

### 1.3 核心技术贡献

1. 提出**自回归视频 - 动作统一世界建模框架**，通过因果注意力与 KV 缓存实现持久长时记忆与严格因果一致性

2. 设计**非对称双流 Mixture-of-Transformers（MoT）架构**，平衡视觉建模能力与动作推理效率

3. 提出**FDM（前向动力学模型）接地的异步推理流水线**，在保持任务性能的同时将执行速度提升 2 倍

4. 引入**噪声历史增强与部分去噪策略**，将视频生成计算量减半且不损失动作预测精度

5. 在 RoboTwin 2.0、LIBERO 等主流基准与真实世界任务中取得 SOTA 性能，仅需 50 个演示即可完成新任务适配

## 3 数据处理流程

### 3.1 统一动作表示规范

为实现跨机器人平台泛化，定义**通用双机械臂动作接口**，总维度 30 维：

- 单臂基础维度：7 维末端执行器（EEF）位姿（XYZ 坐标 + 旋转四元数）+7 维关节角度 + 1 维夹爪开度
- 双臂总维度：(7+7+1)×2=30 维
- 缺失维度处理：用 0 填充，保持统一表示格式
- 动作频率：50Hz（与工业机器人标准控制频率一致）

### 3.2 训练数据集构建

聚合 6 个公开机器人操作数据集 + 内部数据，总规模约**16K 小时**，覆盖多机器人、多场景、多任务：

| 数据集              | 数据特点             | 核心贡献               |
| :------------------ | :------------------- | :--------------------- |
| Agibot              | 移动机械臂多样化操作 | 大规模真实世界场景覆盖 |
| RoboMind            | 多机器人平台演示     | 跨机器人泛化性基础     |
| InternData-A1       | 高保真仿真数据       | sim2real 迁移能力      |
| OXE（OpenVLA 子集） | 多机器人统一基准     | 通用操作能力           |
| UMI Data            | 人类演示数据         | 自然操作范式学习       |
| RoboCOIN            | 双臂操作数据         | 双臂协同控制能力       |

### 3.3 数据预处理与增强流水线

1. **格式统一**：所有数据集转换为`(任务指令, 观测序列, 动作序列)`三元组格式
2. **数据集分割**：每个数据集按 9:1 划分为训练集与验证集，监控训练收敛
3. **动作归一化**：使用训练集的分位数统计对每个动作维度独立归一化
4. **视频下采样**：原始视频从 50Hz 下采样至 12.5Hz（τ=4 倍时间下采样），动作保持 50Hz
5. **序列打包**：将多个短 episode 打包成长序列（最长 10K 令牌），提升 GPU 利用率
6. **噪声历史增强**：训练时以 50% 概率对视频历史令牌注入高斯噪声，噪声强度`s_aug ~ Uniform[0.5, 1.0]`

## 4 模型详细架构设计

### 4.1 整体框架

LingBot-VA 采用**非对称双流自回归扩散架构**，基于流匹配实现连续空间生成：

 **视频流**：负责视觉动力学预测，初始化自 Wan2.2-5B 预训练视频生成模型

- **动作流**：负责动作解码，与视频流同深度但宽度更小
- **融合机制**：通过 Mixture-of-Transformers（MoT）块实现模态间双向交互
- **输出**：未来 K 帧视频隐状态 + 对应 τK 个动作向量

### 4.2 核心组件参数

| 组件         | 参数配置                            | 说明                               |
| :----------- | :---------------------------------- | :--------------------------------- |
| 视频流       | 30 层 Transformer，d_v=3072         | 初始化自 **Wan2.2-5B**，全参数微调 |
| 动作流       | 30 层 Transformer，d_a=768          | 4 倍窄于视频流，新增参数约 350M    |
| 总模型大小   | 5.3B 参数                           | 视频流 5B + 动作流 350M            |
| 视频 VAE     | Wan2.2 因果 **VAE**，压缩比 4×16×16 | **每帧编码为 192 个空间令牌**      |
| 文本编码器   | 冻结 T5 编码器                      | 任务指令通过交叉注意力注入         |
| 动作编解码器 | 单层 MLP，隐藏层 256 维             | 实现动作向量与令牌的双向转换       |
| 位置编码     | RoPE 旋转位置编码                   | 支持长序列建模                     |

### 4.3 Mixture-of-Transformers（MoT）块设计

MoT 块是模态交互的核心，设计原则是**保持模态独立表征的同时实现信息交互**：

1. 视频流与动作流分别计算各自的 Q、K、V 矩阵（使用独立的 QKV 投影层）

2. 动作令牌通过线性层投影至视频维度（d_a → d_v）

3. 所有令牌（视频 + 动作）参与联合自注意力计算

4. 动作令牌通过线性层投影回原维度（d_v → d_a）

5. 视频流与动作流分别添加残差连接与层归一化

   在 LingBot-VA 的 MoT 架构中，视频流和动作流不是单向的 "未来帧→动作"，而是**双向互相条件化**的：

   - 预测未来帧时，会考虑历史动作（动作→未来帧，正因果）
   - 预测动作时，会利用未来帧的信息（未来帧→动作，逆因果）

   这种双向交互让模型的预测更准确：

   1. 视频流知道 "如果我执行这个动作，世界会变成什么样"
   2. 动作流知道 "如果我想让世界变成那样，我该执行什么动作"
   3. 两者在统一的注意力机制中互相传递信息，共同优化

### 4.4 视频稀疏化与令牌交错机制

为平衡视觉建模效率与动作控制频率：

1. 视频帧按 τ=4 倍时间下采样，每 4 个动作对应 1 个视频帧
2. 令牌交错顺序：`[z_t, a_{t,1}, a_{t,2}, a_{t,3}, a_{t,4}, z_{t+1}, a_{t+1,1}, ...]`
3. 预测 K 个视频帧对应生成 4K 个动作，实现 50Hz 高频控制
4. 因果注意力掩码确保每个令牌仅能关注之前的所有令牌

### 4.5 KV 缓存持久化机制

自回归推理时，缓存所有历史令牌的 K、V 值：

- 每次推理仅需计算新令牌的注意力，历史令牌的注意力结果直接复用
- 缓存包含交错的视频与动作令牌，保持完整的轨迹上下文
- 每执行完一个动作块，用真实观测的编码替换缓存中的预测视频令牌
- 有效解决长时任务的时间漂移问题

## 5 训练流程设计

### 5.1 预训练流程

**目标**：学习通用的视觉动力学与动作推理先验

- **训练数据**：16K 小时多源机器人操作数据
- **令牌规模**：1.4T 令牌
- **优化器**：AdamW，权重衰减 0.01，梯度裁剪 2.0
- **学习率**：峰值 1e-4，线性预热 + 余弦退火调度
- **精度**：bfloat16 混合精度训练
- **分类器自由引导**：文本 dropout 率 0.1
- **损失权重**：λ=1（逆动力学损失权重）
- **块大小**：随机采样 K∈[1,4]
- **训练设备**：分布式 GPU 集群（论文未明确具体配置）

### 5.2 后训练（任务适配）流程

**目标**：快速适配新机器人平台与新任务

- **数据需求**：仅需 50 个任务特定演示
- **训练步数**：3K 步（学习率 1e-5，推荐）或 1K 步（学习率 1e-4，快速适配）
- **序列长度**：150,000 令牌
- **冻结策略**：全参数微调
- **收敛判断**：验证集流匹配损失不再下降

### 5.3 关键训练策略

#### 5.3.1 教师强制训练

- 将交错的视频 - 动作序列视为统一语言模型序列
- 训练时使用真实令牌作为上下文预测下一个令牌
- 通过因果注意力掩码确保严格的时间因果性
- 与机器人部署时的真实观测反馈天然匹配，无训练 - 测试分布偏移

#### 5.3.2 可变块大小训练

- 训练时随机采样块大小 K∈[1,8]
- 模型学习生成不同时间长度的连贯预测
- 推理时可自由选择 K 值：大 K 减少自回归步骤，小 K 增加闭环修正频率
- 部署默认 K=4，平衡计算效率与鲁棒性

#### 5.3.3 噪声历史增强

训练时以 50% 概率对视频历史令牌注入噪声：

z~≤t={(1−saug)ϵ+saugz≤t,z≤t,p=0.5,saug∈[0.5,1],ϵ∼N(0,I)1−p=0.5

- 训练逆动力学模型从部分噪声的视频表示中预测动作
- 推理时可仅将视频去噪至 s=0.5~0.6，大幅减少计算量

## 6 推理流程设计

### 6.1 同步推理算法（算法 1）

**适用场景**：冷启动阶段或对延迟要求不高的任务

```
def sync_inference(o0, K):
    z0 = video_vae.encode(o0)
    kv_cache = {z0}
    t = 0
    while True:
        # 生成视频块（部分去噪至s=0.6）
        eps_v = torch.randn_like(z0)
        z_tilde = eps_v + 0.5 * integrate(video_flow, eps_v, 0, 0.6, kv_cache)
        
        # 生成动作块（全去噪至s=1.0）
        eps_a = torch.randn_like(action_token)
        a_chunk = eps_a + integrate(action_flow, eps_a, 0, 1.0, kv_cache + z_tilde)
        
        # 执行动作并收集真实观测
        obs_chunk = []
        for a in a_chunk:
            o_next = robot.execute(a)
            obs_chunk.append(o_next)
        
        # 更新KV缓存
        z_chunk = [video_vae.encode(o) for o in obs_chunk]
        kv_cache.update(z_chunk + a_chunk)
        t += K
```

### 6.2 FDM 接地异步推理算法（算法 2）

**核心优势**：将预测与执行并行，隐藏推理延迟，速度提升 2 倍

```
import threading
from queue import Queue

def async_inference(o0, K):
    z0 = video_vae.encode(o0)
    kv_cache = {z0}
    obs_queue = Queue()
    
    # 冷启动：预测初始动作块
    z_tilde_1K, a_0K_1 = predict_next_chunk(kv_cache)
    t = 0
    
    def execution_thread():
        while True:
            for a in current_a_chunk:
                o_next = robot.execute(a)
                obs_queue.put(o_next)
    
    # 启动执行线程
    threading.Thread(target=execution_thread, daemon=True).start()
    
    while True:
        # 获取真实观测并更新缓存
        if t > 0:
            obs_chunk = [obs_queue.get() for _ in range(K)]
            z_chunk = [video_vae.encode(o) for o in obs_chunk]
            kv_cache.update(z_chunk + last_a_chunk)
        
        # FDM接地：用真实观测重对齐当前视觉状态
        tmp_cache = kv_cache + current_a_chunk
        z_current = fdm.predict(tmp_cache)
        tmp_cache.update(z_current)
        
        # 预测下一个动作块
        z_next, a_next = predict_next_chunk(tmp_cache)
        
        # 更新当前执行的动作块
        current_a_chunk = a_next
        last_a_chunk = current_a_chunk
        t += K
```

### 6.3 部分去噪加速策略

- 视频令牌：仅去噪至 s=0.6，使用 3 步欧拉求解器
- 动作令牌：全去噪至 s=1.0，使用 10 步欧拉求解器
- CFG 缩放：视频 CFG=5.0，动作 CFG=1.0（减少计算量）
- 推理精度：bfloat16，支持 TensorRT 加速

## 7 损失函数设计

所有损失均基于**流匹配**框架，学习从噪声到目标数据的连续向量场。

### 7.1 流匹配基础

流匹配定义一个时间依赖的向量场vs，描述粒子从噪声ϵ到目标数据x1的流动轨迹：

dsdx(s)=vs(x(s)),x(0)=ϵ∼N(0,I)

模型通过最小化预测向量场与真实向量场的 MSE 损失进行训练：

LFM=Es,ϵ,x1[vθ(x(s),s)−x˙(s)2]

其中x˙(s)=x1−ϵ为真实速度场。

### 7.2 视频动力学损失（Ldyn）

监督视频流预测未来视觉状态的能力：

Ldyn=Et,s,zt+1,ϵ[vθ(zt+1(s),s,z~≤t,a<t∣c)−z˙t+1(s)2]

- zt+1(s)=(1−s)ϵ+szt+1：噪声与真实视频隐状态的插值
- z~≤t：可能经过噪声增强的历史视频隐状态
- c：任务指令的 T5 编码

### 7.3 逆动力学损失（Linv）

监督动作流从视觉状态转换解码动作的能力：

Linv=Et,s,at,ϵ[vψ(at(s),s,z≤t+1,a<t∣c)−a˙t(s)2]

- at(s)=(1−s)ϵ+sat：动作的噪声插值
- z≤t+1：当前与未来视频隐状态（可能含噪声）

### 7.4 前向动力学损失（Lfdm）

仅用于异步推理的 FDM 模块训练，监督模型基于当前状态与动作预测下一状态的能力：

Lfdm=Et,s,z^t+1,ϵ[vψ(z~t+1,s,zt,at,z~<t,a^<t∣c)−z˙t+1(s)2]

### 7.5 总损失函数

L=Ldyn+λLinv

- 预训练时λ=1
- 后训练时可根据任务特性调整λ值

## 9 实时部署方案

### 9.1 推理加速技术栈

1. **KV 缓存加速**：计算量从 O (n²) 降至 O (n)，推理速度提升 3-5 倍
2. **部分去噪**：视频生成步骤减半，速度提升 2 倍
3. **模型量化**：支持 bfloat16/int8 量化，显存占用减少 40-50%
4. **TensorRT 优化**：算子融合与内核优化，推理速度提升 2-3 倍
5. **分布式推理**：支持多 GPU 并行推理，满足高并发要求

### 9.2 部署架构

```
用户指令 → 文本编码器 → LingBot-VA 推理引擎 → 动作解码器 → 机器人控制器
                          ↑
                          |
                      真实观测 ← 机器人传感器
```

- 推理引擎与机器人控制器通过 ROS/ROS2 通信
- 观测与动作的延迟控制在 20ms 以内
- 支持实时任务中断与重规划
