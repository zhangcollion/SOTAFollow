## 1️⃣ 一句话评价

LingBot-Map 首次提出**几何上下文注意力（GCA）**机制，通过3D空间距离约束替代全连接注意力，将流式3D重建的精度与速度解耦——在 ETH3D 取得 **98.98% F1**，同时维持 **20 FPS** 实时处理，突破了过去流式方法"长序列必漂移"的核心瓶颈。

---

## 2️⃣ Know How（关键工程诀窍）

### 2.1 为什么全连接注意力无法处理流式重建？

传统 Transformer 的注意力是 $O(N^2)$ 全连接——当前帧的 $N$ 个锚点要与历史**所有** $M$ 个锚点交互。当处理 10,000 帧时，$M$ 会膨胀到数十万级别，显存爆炸且帧率暴跌。

**核心矛盾**：我们需要"看见"很远的历史（用于闭环修正），但又无法承担全连接的计算代价。

### 2.2 核心 insight：3D空间距离 = 天然稀疏性

现实物理世界给了我们一个关键先验：**3D空间中，只有距离近的物体才可能匹配**。相机当前位置周围2米内的特征点，才可能是闭环候选；10米外的信息几乎不可能与当前帧重合。

这意味着我们可以**用3D距离代替注意力分数**作为筛选依据——把 $O(N^2)$ 稠密计算，转化为查询半径 $r$ 内的**稀疏计算**。

### 2.3 三种上下文的分工设计

| 上下文 | 筛选方式 | 负责任务 |
|--------|----------|---------|
| **锚点上下文** | 当前帧自身 | 提供坐标和尺度的全局基准 |
| **位姿参考窗口** | FIFO滑动窗口（最近$T$帧） | 局部几何平滑，短时相机运动 |
| **轨迹记忆库** | 关键帧筛选（距离+旋转+视觉） | 长程漂移修正，闭环检测 |

### 2.4 关键帧筛选的门控机制

如果仅用"平移距离"作为唯一标准，原地转圈时（位移<阈值）会导致：
- 全局记忆库对房间其他角度"失明"
- 相机移动时找不到匹配特征 → 地图错位

**解决**：综合**完整相机位姿**（平移+旋转），确保任何显著视角变化都被记录。

## 4️⃣ 算法详细方法解析

### 4.1 整体架构

```
输入视频流（RGB图像）
    ↓
┌─────────────────────────────────────────────────────────────┐
│  2D特征提取（ConvNet/ViT backbone）                          │
│  输入：H×W×3  →  输出：H/8 × W/8 × C 特征图                     │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  单目深度估计网络（Monocular Depth Network）                     │
│  输出：H/8 × W/8 × 1 深度图                                    │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  3D特征提升（Lift 2D to 3D）                                  │
│  2D特征 + 深度 → 带3D坐标的锚点特征                             │
│  锚点数量：H/8 × W/8（如256×256 → 32×32 = 1024个锚点）          │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  几何上下文注意力（GCA）                                        │
│  三种上下文交互：锚点 + 位姿窗口 + 轨迹记忆库                      │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│  3D重建输出 + 相机位姿估计                                      │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 几何上下文注意力（GCA）核心公式

**传统 Transformer 注意力**：
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V$$

**GCA（带几何掩码）**：
$$\text{Attention}(Q, K, V, M) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}} + M\right)V$$

其中几何掩码 $M$ 定义为：
$$M_{ij} = \begin{cases} 0 & \text{if } \|p_i - p_j\| < r \\ -\infty & \text{if } \|p_i - p_j\| \geq r \end{cases}$$

- $p_i, p_j$：查询点和键点的3D坐标
- $r$：空间搜索半径（如2.0米）
- 只有落在半径 $r$ 内的锚点对才能交互

### 4.3 三种上下文构建

#### 位姿参考窗口（滑动窗口）
- **维护方式**：FIFO 队列，容量为最近 $T$ 帧
- **筛选逻辑**：时间驱动，最新 $T$ 帧全部保留
- **作用**：确保局部几何平滑，处理短时相机运动

#### 轨迹记忆库（关键帧）
- **筛选条件**（需同时满足或满足其一）：
  | 条件 | 阈值 | 说明 |
  |------|------|------|
  | 平移距离 | > 0.5m | 相对上一个关键帧的位移 |
  | 旋转角度 | > 15° | 相机朝向变化 |
  | 视觉变化 | > 阈值 | 特征多样性 |

- **3D距离查询**：给定当前帧锚点，只在半径 $r$ 内提取历史锚点

#### 锚点上下文
- 当前帧的3D锚点集合
- 提供坐标和尺度的全局基准

### 4.4 训练策略

```python
# 总损失
total_loss = reconstruction_loss + depth_loss + consistency_loss

# 重建损失：3D点云重建误差
reconstruction_loss = ||P_pred - P_gt||_1

# 深度损失：深度图监督
depth_loss = ||D_pred - D_gt||_1

# 一致性损失：相邻帧几何一致性
consistency_loss = ||T_{i→j} - T_{gt}||_robust
```

## 7️⃣ 附录细节

### A.1 实现细节

**GCA 模块配置：**
- 特征维度：256
- 注意力头数：8
- 搜索半径：2.0米
- 位姿窗口容量：10帧

**网络结构：**
```python
class GeometricContextAttention(nn.Module):
    def __init__(self, feature_dim=256, num_heads=8, search_radius=2.0):
        self.q_proj = nn.Linear(feature_dim, feature_dim)
        self.k_proj = nn.Linear(feature_dim, feature_dim)
        self.v_proj = nn.Linear(feature_dim, feature_dim)
        self.out_proj = nn.Linear(feature_dim, feature_dim)

    def forward(self, query_coord, key_coord, query_feat, key_feat):
        # 1. 3D距离掩码
        distances = torch.cdist(query_coord, key_coord)
        geom_mask = torch.where(distances >= self.search_radius,
                               float('-inf'), 0.0)

        # 2. 多头注意力 + 掩码
        q = self.q_proj(query_feat)
        k = self.k_proj(key_feat)
        v = self.v_proj(key_feat)

        attn = F.softmax((q @ k.transpose(-2,-1))/math.sqrt(d) + geom_mask, dim=-1)
        return self.out_proj(attn @ v)
```

### A.2 关键帧筛选阈值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 平移阈值 | 0.5m | 触发关键帧的最小位移 |
| 旋转阈值 | 15° | 触发关键帧的最小旋转角 |
| 位姿窗口大小 | 10帧 | FIFO队列容量 |
| 搜索半径 | 2.0m | GCA空间查询半径 |

## 8️⃣ 总结

LingBot-Map 提出了**几何上下文注意力（GCA）**机制，核心贡献：

1. **3D距离掩码**：用物理空间约束替代全连接注意力，$O(N^2)$ → $O(N \cdot K)$
2. **三种互补上下文**：锚点（基准）+ 位姿窗口（平滑）+ 轨迹记忆（修正）
3. **端到端流式学习**：联合优化深度、特征、注意力，兼顾速度与精度

在 ETH3D 取得 **98.98% F1**，同时维持 **20 FPS** 实时处理，首次证明了流式重建可以在长序列（10,000+帧）上同时实现高精度与低延迟。

> **核心价值**：将"3D空间距离"这一物理先验引入注意力机制，为实时稠密重建提供了一个可扩展的架构范式。
