## 一句话总结

**π0.7 通过多样化上下文提示（Diverse Context Conditioning）策略，让单一通用机器人基础模型在无需任务特定微调的情况下，在灵巧操控、指令跟随、跨本体迁移和组合泛化等多个维度达到或接近专用 RL 微调模型的水平。**

---

## 背景与问题动机

### 1.1 现状与挑战

传统视觉-语言-动作模型（VLA）面临的核心困境：

- **组合泛化能力不足**：语言模型可以灵活组合不同知识解决新问题，但机器人 VLA 难以做到——不仅无法解决新任务，甚至连训练数据中的任务都无法流畅执行
- **数据多样性 vs 质量的两难**：使用大规模多样化数据（含次优演示、自主演出失败案例、非机器人数据）会因策略差异导致模型在多模态间"平均化"，反而降低性能
- **任务特定微调的必要性**：当前最佳性能的策略往往需要针对下游任务进行微调，失去了通用基础模型的意义

### 1.2 核心洞察

作者的核心洞察来自图像/视频生成领域的"Prompt Expansion"思想：**详细且丰富的上下文信息不仅告诉模型"要做什么"（What），还能告诉"如何做"（How），从而消歧大方异构数据中的策略差异**。

通过在训练时为每个样本注入多模态上下文（语言、策略元数据、子目标图像），模型能够：
1. 从次优数据中学习而不损害性能
2. 利用多样化数据驱动组合泛化
3. 在测试时灵活配置提示，实现模型"可操纵"（Steerable）

## 方法详解

### 2.1 整体架构

**关键组件：**

| MEM History Encoder | — | 将任意数量历史帧压缩至固定 token 数 |

### 2.2 训练目标

标准的 VLA 训练目标是最大化条件似然：

$$
\max_\theta \mathbb{E}_{\mathcal{D}} \left[ \log \pi_\theta(a_{t:t+H} \mid o_{t-T:t}, \mathcal{C}_t) \right]
$$

其中 $\mathcal{C}_t$ 是 **Context（上下文）**，传统方法仅包含语言指令 ℓ_t。

π0.7 将 Context 扩展为多模态形式：$\mathcal{C}_t = \{\ell_t, \hat{\ell}_t, g_t, m, c\}$

### 2.3 多模态 Context 详解

#### 2.3.1 子任务指令（Subtask Instructions）

除整体任务描述 ℓ_t（如 "clean the kitchen"）外，加入中间层语义子任务 $\hat{\ell}_t$（如 "open the fridge door"）。在推理时可由高层策略或人类动态生成，实现 step-by-step Verbal Coaching。

#### 2.3.2 子目标图像（Subgoal Images）

语言难以描述的细节（如机械臂如何抓取把手），由 **未来帧图像** 提供视觉化目标。推理时由轻量级世界模型生成：

$$
\max_\psi \mathbb{E}_{\mathcal{D}_g} \left[ \mathcal{L}_{\text{CFM}}(g^*_t, g_\psi(o_t, \hat{\ell}_t, m)) \right]
$$

其中 $g^*_t = o_{t_{\text{end}}}$ 是 segment 末尾的真实图像作为 ground truth。

世界模型从 **BAGEL**（14B 图文生成模型）初始化，在推理时以异步方式每 4 秒或语义意图变化时重新生成。

#### 2.3.3 Episode 元数据（Episode Metadata）

用于消歧异构数据质量的**策略标签**：

| 元数据字段 | 含义 | 作用 |
|-----------|------|------|
| **Overall Speed** | Episode 长度（以 500 步为区间离散化） | 关联速度与质量 |
| **Overall Quality** | 任务执行质量分（1-5 分，5 最高） | 指示最优行为 |
| **Mistake** | 是否在 segment 内犯错（抓取失败、做错子任务等） | 过滤失败案例 |

**关键设计**：在训练时随机 dropout 各部分上下文（子目标图像 25%、子任务指令 30%、Episode 元数据 15% 整体+各字段 5% 单独 dropout），使模型在测试时可灵活使用任意子集的提示。

#### 2.3.4 控制模式（Control Mode）

文本标识 $c \in \{\text{joint}, \text{ee}\}$ 区分关节空间与末端执行器空间控制。

### 2.4 完整提示示例

```
<多视角观察><多视角子目标>
Task: peel vegetables. 
Subtask: pick up the peeler. 
Speed: 8000. 
Quality: 5. 
Mistake: false. 
Control Mode: joint.
<本体感受>
```

### 2.5 训练数据

π0.7 的训练数据构成极为多样：

1. **演示数据**：多机器人平台（静态/移动、单臂/双臂）、多种环境（实验室/家庭/真实家庭）
2. **自主数据**：大规模策略评估的 rollout、RL 后训练模型的数据（含失败案例）
3. **人类 egocentric 视频数据**
4. **互联网多模态数据**：目标定位、属性预测、VQA、图文任务
5. **开源机器人数据集**（Open X-Embodiment 等）

**重要创新**：重度使用次优数据。通过 episode metadata 的条件化，模型可从失败中学习而不损害最终性能——这相当于将 RL 专用策略的技能"蒸馏"到通用模型中。

### 2.6 推理流程（Algorithm 1）

```
输入: 初始观察 o0, 任务指令 ℓ, Episode 元数据 m, 控制模式 c
1: 初始化子任务 ˆℓ（来自高层策略或人类 coaching）
2: g* ~ p_ψ(g* | o0, ˆℓ, m)
3: C = {ℓ, ˆℓ, g*, m, c}
4: a_{t:t+H} ~ π_θ(a | o_{t-T:t}, C)     ▷ 可选 CFG
5: for t = 0, 1, 2, ... do
6:     if ˆℓ 变化 or Δ 秒计时器到期 then
7:         g* ~ p_ψ(g* | o_t, ˆℓ, m)     ▷ 非阻塞异步
8:         C = {ℓ, ˆℓ, g*, m, c}
9:     end if
10:    if ˆH 步已执行 then
11:        a_{t:t+H} ~ π_θ(a | o_{t-T:t}, C, a_{t-ˆH:})  ▷ RTC 异步
12:    end if
13:    执行 a_t
14: end for
```

推理使用 **异步执行**：子目标图像和子任务指令生成在独立线程运行，VLA 始终使用最新可用结果。

### 2.7 Classifier-Free Guidance（CFG）

通过 CFG 增强动作质量：

$$
\nabla_a \log \pi_\theta(a_{t:t+H} \mid o_t, \mathcal{C}_t) + \beta (\nabla_a \log \pi_\theta(a_{t:t+H} \mid o_t, \mathcal{C}_t^{\text{uncond}}) - \nabla_a \log \pi_\theta(a_{t:t+H} \mid o_t, \mathcal{C}_t))
$$

实验中 CFG 权重 β ∈ {1.3, 1.7, 2.2}，主要应用于 episode metadata 以激发灵巧任务的高性能。

## 实验结果

### 3.1 开箱即用灵巧操控性能

π0.7 在无需任何任务特定微调的情况下，在多种复杂灵巧任务上匹配或超过专用 RL 微调模型：

| 任务 | π0.7 vs π*₀.6 RL Specialist | 结论 |
|------|---------------------------|------|
| Laundry (T-Shirts & Shorts) | 更高吞吐量 | **超越** |
| Laundry (Diverse - Hardest) | 更高吞吐量 | **超越** |
| Make Espresso | 相当 | **持平** |
| Box Building | 更高吞吐量 | **超越** |
| Make PB Sandwich | 相当 | **持平** |
| Slice Zucchini | 相当 | **持平** |
| Peel Fruits/Vegetables | 相当 | **持平** |
| Take Out Trash | 相当 | **持平** |

**消融实验**：移除 episode metadata 或移除评估数据均导致显著性能下降，证明两者对复杂灵巧任务至关重要。

### 3.2 指令跟随能力

在 4 个未见的厨房和 2 个未见的卧室环境中，π0.7 对 3-6 步指令序列的跟随成功率显著超越 π0.5 和 π0.6。

**打破数据偏见**：在"Reverse Bussing"和"Reverse Fridge to Microwave"等违背训练数据分布的任务中，π0.7 仍能正确跟随指令，证明其语言跟随能力已足够强到克服数据中的自然偏见。

### 3.3 跨本体迁移（Cross-Embodiment Transfer）

**最引人注目的结果之一**：在衬衫折叠任务上，从小型静态双臂机器人迁移到大型 UR5e 双臂机器人：

| 对比 | Task Progress | Success Rate |
|------|--------------|--------------|
| **π0.7 (GC)** | 85.6% | **80%** |
| **人类专家 teleoperator** | 90.9% | 80.6% |

π0.7 的表现与具有丰富操作经验的人类专家相当，而这是**零样本迁移**——模型从未在 UR5e 上见过任何折叠演示！

更关键的是，模型并非盲目复制源机器人策略，而是**自适应发现新策略**：
- 源机器人用双臂配合开袋，UR5e 改用单臂 pick-and-place
- 源机器人倾斜末端执行器抓取，UR5e 改用垂直抓取（更适合其臂形）

### 3.4 组合任务泛化

**语言 Coaching**：π0.7 可以通过人类 step-by-step 口头指令来"教会"新任务（如用空气炸锅烹饪）。这些 coaching 数据进一步训练高层语言策略，实现完全自主执行。

实验验证了 5 个新长周期任务（Loading/Unloading Air Fryer, Toasting Bagel 等），通过 coaching 数据训练的 autonomous 策略性能与 live coaching 持平。

### 3.5 数据规模与多样性缩放分析

**核心发现**：
- **有 metadata 的 π0.7** 在数据集越大（即使平均质量下降）时持续提升
- **无 metadata 的 π0.7** 在加入更多低质量数据后性能反而下降
- 移除最高任务多样性的 20% 数据会导致所有短周期未见任务性能显著下降

这证明了多样化提示策略使模型设计更具**可扩展性**：受益于更大规模数据集，即使其中混杂大量次优数据。

## 附录重要结论

### Appendix C: 世界模型训练细节

- 初始化自 BAGEL（14B MoE），保留其图文理解和编辑能力
- 使用高质量分段语言标签的机器人数据 + egocentric 人类视频数据
- 额外混入开源图像编辑和视频数据集保持语义知识
- 每个训练样本包含：subtask instruction ˆℓ、3 路相机输入 o_t、3 张目标图像 o_{t_end}
- 相机输入同时用 ViT（448×336）和 VAE（512×384）处理（patch size 差异 14 vs 16）
- 推理时每 Δ=4 秒重新生成子目标（与 SuSIE 一致）

### Appendix D: 推理优化

- 单 GPU NVIDIA H100，最小配置推理延迟 38ms（3 相机输入 + 5 denoising 步 + 训练时 RTC）
- 启用 MEM encoder + 子目标图像：最坏情况 127ms
- 子目标图像生成：4×H100 张量并行 + 8-bit 量化 + SageAttention，25 步去噪在 1.25 秒内完成
- 推理采用朴素异步策略：π0.7 执行动作时，世界模型并行生成下一子目标

### Appendix E: 关节空间 vs 末端执行器控制

跨本体任务实验表明，末端执行器控制在性能上无明显优势，因此主实验均采用关节空间控制。

### Appendix F: 人类受试者研究

- 招募 10 名经验最丰富操作员（top 2%，平均操作经验 ~375 小时）
- 所有操作员均未曾在 UR5e 上进行过衬衫折叠，构成与 π0.7 相同的零样本设置
- 每人进行 3 次试验，共 30 次试验
- 结果：人类 90.9% task progress / 80.6% success rate；π0.7 (GC) 85.6% / 80%
- 结论：π0.7 在跨本体衬衫折叠任务上达到人类专家水平

### 核心贡献

1. **多样化上下文条件化（Diverse Context Conditioning）**：将提示从单一语言指令扩展为多模态上下文（语言子任务、策略元数据、子目标图像），使模型能从异构数据中学习而不产生平均化效应

2. **Episode Metadata 作为策略条件化信号**：用质量/速度/错误标签消歧次优数据，实现从 RL 专用策略到通用模型的"蒸馏"

3. **零样本跨本体迁移**：在灵巧任务（衬衫折叠）上，模型自主发现适应目标本体的操控策略，达到人类专家水平

4. **语言 Coaching 范式**：利用强大的指令跟随能力，通过口头 step-by-step 指导教会新任务，无需额外演示数据

### 局限性

- 零样本泛化成功率（60-80%）仍低于分布内任务（>90%）
- 真实世界泛化能力的衡量困难——数据集中可能已包含相关技能只是标签不同
- 子目标图像生成依赖 14B 世界模型，推理延迟和计算成本较高

### 个人点评

**这篇论文最值得关注的 insight** 是：多样化的上下文提示本质上将"如何做"的策略信息从数据中分离出来，使同一数据可以教模型多种策略，模型在测试时通过上下文选择策略。这与语言模型中 system prompt 的作用异曲同工，只是扩展到了机器人控制的多模态场景。

π0.7 展示了 Physical Intelligence 在机器人基础模型上的持续迭代能力——从 π0 到 π0.7，每一代都有明确的能力提升路径。其"通用模型匹配专用 RL"的核心主张，在实践层面有重要意义：未来或许可以用通用模型作为起点，通过语言或少量示范快速适配新任务，而非每次都从零训练专用策略。
