## 1. Motivation（问题背景）

### 1.1 什么是机器人 Foundation Model？

**图1：机器人 Foundation Model 定位**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    机器人智能层次结构                                    │
│                                                                     │
│   L4: 通用具身智能 (General Embodied Intelligence)                  │
│       —— 长期自主学习、跨任务泛化、终身学习                           │
│                                                                     │
│   L3: 机器人 Foundation Model + 领域微调                             │
│       —— 大规模预训练 + 下游任务适应（当前演讲重点）                 │
│                                                                     │
│   L2: 任务专用策略 (Task-Specific Policy)                           │
│       —— 单任务 BC/RL 策略                                          │
│                                                                     │
│   L1: 经典控制 (Classical Control)                                  │
│       —— PID / MPC / 轨迹规划                                       │
└─────────────────────────────────────────────────────────────────────┘
```

Sergey Levine 在演讲中指出，机器人 Foundation Model 的目标是**从大规模、多样化的数据中学习可泛化的策略**，类似于 NLP 中的 GPT —— 预训练一个大模型，在下游任务上微调即可。

### 1.2 第一代 VLA 的局限性（RT-1 / RT-2）

| 方法 | 核心思路 | 局限性 |
|------|---------|--------|
| **RT-1** | FiLM-conditioned RNN 融合图像 + 语言指令 | 数据量有限（13k demos），泛化能力不足 |
| **RT-2** | VLA 直接输出动作 token（Vision-Language-Action） | 依赖大规模互联网数据训练，推理成本极高 |
| **RT-X** | Open Vocabulary Manipulation | 泛化到新物体但仍受限于数据分布 |

Levine 指出第一代 VLA 的核心问题：
- **数据瓶颈**：真实机器人示教数据稀缺，难以覆盖长尾场景
- **泛化不足**：在未见过的物体/场景上性能骤降
- **物理常识缺失**：无法理解"柔软物体要轻拿轻放"这类常识

### 1.3 第二代 VLA 的核心洞察

**核心问题：如何让机器人像 GPT 那样泛化到开放世界？**

Levine 的核心洞察是：

> **"The key is not the architecture, but the data."**
> **—— 关键不在于模型架构，而在于数据的多样性和规模。**

这解释了为什么 π₀ 系列（Physical Intelligence）采用：
1. **Diffusion Policy + VLA 架构**
2. **强化学习后训练（RL Post-training）** 而非纯模仿学习
3. **跨形态数据**（不同机器人、不同场景）

## 2. 一句话总结

> **Sergey Levine 在 Actuate 2025 上系统梳理了机器人 Foundation Model 的演进路径：互联网视频预训练提供物理常识，跨形态数据实现泛化，VLA 架构从"感知-动作"端到端映射，第二代 VLA 的核心在于用强化学习后训练（RL Post-training）弥补模仿学习的数据分布缺陷。**

## 4. 核心内容解析

### 4.1 互联网视频作为物理常识来源

**图2：互联网视频预训练的价值**

```
┌─────────────────────────────────────────────────────────────────────┐
│                 物理世界知识蒸馏路径                                   │
│                                                                     │
│   YouTube / TikTok / 开源视频                                        │
│          ↓                                                           │
│   【海量物理交互视频】                                               │
│   —— 物体交互、碰撞、拿起、放置、工具使用                           │
│          ↓                                                           │
│   视频表征学习（Video Representation Learning）                       │
│   —— 动作语义、时间因果、物理直觉                                    │
│          ↓                                                           │
│   预训练的视觉-语言模型                                              │
│   —— 理解"拿起"="lift"、"打开"="open"                              │
│          ↓                                                           │
│   机器人控制策略                                                    │
│   —— 将语言指令映射到物理动作                                        │
└─────────────────────────────────────────────────────────────────────┘
```

Levine 认为互联网视频是**被忽视的物理常识来源**：
- 人类演示视频包含大量物理交互知识
- 但视频本身是第三人称观察，不能直接用于机器人控制
- 需要将其转化为**可操作的表征**（Actionable Representation）

### 3.2 Cross-Embodiment：跨形态数据融合

**图3：跨形态数据的重要性**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Cross-Embodiment 数据策略                          │
│                                                                     │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ 双臂机器人│  │ 四足机器人│  │ 机械臂   │  │ 无人机   │          │
│  │ (Franka) │  │ (Spot)   │  │ (UR5)    │  │ (DJI)    │          │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘          │
│       │              │              │              │                │
│       └──────────────┴──────────────┴──────────────┘                │
│                               ↓                                      │
│              【动作抽象层 Action Abstraction】                        │
│              —— 抓住、放置、推动、旋转（任务级别）                   │
│                               ↓                                      │
│                    跨形态泛化策略                                     │
│                    —— 同一策略可控制不同形态                          │
└─────────────────────────────────────────────────────────────────────┘
```

Levine 在 BC-Z 项目中验证了跨形态学习的可行性：
- 将"把苹果放到碗里"这个任务从双臂机器人迁移到单臂机械臂
- **关键抽象**：动作语义而非具体执行方式

### 3.3 第二代 VLA 架构：π₀ 的启示

**图4：π₀ 架构（Physical Intelligence）**

```
┌──────────────────────────────────────────────────────────────────────┐
│                         π₀ 架构（简化版）                              │
│                                                                       │
│  输入层：                                                              │
│  ┌──────────────┐  ┌──────────────┐                                 │
│  │ RGB 图像     │  │ 语言指令      │                                 │
│  │ (视觉观察)   │  │ (任务描述)   │                                 │
│  └──────┬───────┘  └──────┬───────┘                                 │
│         │                  │                                          │
│         ↓                  ↓                                          │
│  ┌──────────────────────────────────────────┐                       │
│  │       视觉-语言编码器 (ViT + LLM)         │                       │
│  │  融合图像和语言表征，输出统一表征           │                       │
│  └────────────────────┬─────────────────────┘                       │
│                      │                                               │
│         ┌────────────┴────────────┐                                 │
│         ↓                         ↓                                 │
│  ┌──────────────────┐   ┌──────────────────┐                       │
│  │  扩散策略头       │   │  RL 后训练        │                       │
│  │  (Diffusion Head)│   │  (强化学习微调)   │                       │
│  │  输出动作序列     │   │  提升长尾任务性能 │                       │
│  └──────────────────┘   └──────────────────┘                       │
└──────────────────────────────────────────────────────────────────────┘
```

π₀ 的核心创新：
1. **扩散模型作为动作头**（Diffusion Policy 范式）—— 更稳定的动作预测
2. **RL Post-training** —— 解决模仿学习在长尾任务上的失败
3. **互联网视频预训练** —— 注入物理常识

### 3.4 强化学习后训练的意义

Levine 强调了一个关键洞察：

> **"Imitation learning can only replicate what it has seen. Reinforcement learning can discover what has never been demonstrated."**
> **—— 模仿学习只能复现见过的行为，强化学习能发现从未被示教过的行为。**

**图5：RL Post-training 解决的难题**

```
模仿学习局限：
┌────────────────────────────────────┐
│  示教数据分布 ←→ 长尾任务失败        │
│  数据稀疏 → 泛化不足                 │
└────────────────────────────────────┘

RL Post-training 解决思路：
┌────────────────────────────────────┐
│  Step 1: 用模仿学习学到基础技能      │
│          （快速收敛，但有上限）       │
│                                     │
│  Step 2: 用 RL 探索长尾动作空间      │
│          （突破示教数据的局限）       │
│                                     │
│  Step 3: 奖励塑形（Reward Shaping）│
│          —— 设计引导探索的奖励函数   │
└────────────────────────────────────┘
```

## 4. 关键要点（KnowHow）

### 4.1 数据为王

Levine 在演讲中反复强调：**"Data is more important than architecture."**

| 数据策略 | 具体做法 |
|---------|---------|
| **跨形态数据** | 收集不同机器人、不同场景的演示数据 |
| **互联网视频** | 利用 YouTube 等视频学习物理交互常识 |
| **模拟器数据** | 生成大规模合成数据（域随机化） |
| **人机协作标注** | 示教 + 偏好反馈（Preference Feedback） |

### 4.2 物理常识的注入方式

互联网视频作为预训练信号的价值：
- **动作语义理解**："搅拌"vs"摇晃"vs"混合"
- **时间因果推理**：先拿起、后放下、再合上
- **物理直觉**：柔软物体需要轻力、刚性碰撞有反弹

### 4.3 VLA 训练的两阶段范式

```
Stage 1: 大规模预训练
├── 互联网视频 → 表征学习
├── 跨形态机器人数据 → 基础动作能力
└── 语言指令对齐 → 任务理解

Stage 2: 任务微调
├── 少量示教数据 → 快速适应
├── RL Post-training → 长尾探索
└── 奖励塑形 → 精确控制
```

### 4.4 对自动驾驶的启示

Levine 的研究路线对自动驾驶有重要启示：

| 机器人 VLA 洞察 | 自动驾驶对应 |
|---------------|-------------|
| 跨形态数据 | 多传感器融合（Camera + LiDAR + Radar） |
| 互联网视频预训练 | 驾驶视频预训练（DashCam Video） |
| RL Post-training | 强化学习微调（Human-in-the-loop RL） |
| 物理常识注入 | 世界模型预训练（World Model Pretraining） |

## 8. 核心总结

### 8.1 第二代 VLA 的三大支柱

```
┌────────────────────────────────────────────────────────────────────┐
│                    第二代 VLA 技术栈                                  │
│                                                                    │
│   ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐   │
│   │  互联网视频预训练 │  │  跨形态数据融合  │  │  RL Post-training│   │
│   │  (物理常识)      │  │  (泛化能力)      │  │  (长尾探索)      │   │
│   └────────┬────────┘  └────────┬────────┘  └────────┬────────┘   │
│            │                    │                    │              │
│            └────────────────────┴────────────────────┘              │
│                                 ↓                                   │
│                    通用机器人 Foundation Model                      │
│                    （可泛化、可适应、可探索）                          │
└────────────────────────────────────────────────────────────────────┘
```

### 8.2 关键洞察

1. **"数据 > 架构"**：在 VLA 领域，数据的多样性和规模比模型架构创新更重要
2. **"RL 是模仿学习的补充而非替代"**：两者结合才能覆盖从常见到长尾的任务分布
3. **"物理常识来自互联网"**：视频预训练是将互联网知识迁移到机器人的最有效途径
4. **"Cross-embodiment 是泛化的关键"**：跨形态数据让策略学会抽象的动作语义

### 8.3 对从业者的建议

| 场景 | 建议 |
|------|------|
| **数据收集** | 优先收集多样化场景数据，而非单一任务大量数据 |
| **模型选型** | 先用现有 VLA 架构（如 π₀），避免重复造轮子 |
| **训练策略** | 模仿学习打基础 + RL Post-training 突破长尾 |
| **评估指标** | 不仅看平均性能，更要看长尾任务（≥10 个未见场景） |
